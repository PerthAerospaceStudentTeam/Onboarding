# Serial Communication
Serial communication is the process of streaming data over a data bus by sending each bit sequentially. The hardware interface can operate using a single wire, but some protocols use up to four. In contrast, parallel communcation will use a bus of eight, sixteen or more wires to transfer all multiple data bits at once. In embedded systems serial I/O is generally preferred due to space requirements and generally being fast enough for most applications. Commonly encountered serial standards include: 
- Universal Asynchronous Receiver/Transmitter (UART)
- Inter-Integrated Circuit (I2C) 
- Serial Peripheral Interface (SPI) 
- Universal Serial Bus (USB)
- Ethernet 

## Asynchronous Serial Communication 
Asynchronous serial protocols facilitate communication without the use of an external clock signal, reducing the number of wires required to physically transfer the data. It does, however, introduce additional overhead to insure data is transferred reliably. 

## Synchronous Serial Communication
As expected, synchronous serial protocols include a clock line alongside their data lines. Ensuring all devices using the bus to communicate share a clock signal. I2C and SPI are two of the most common synchronous interfaces you will encounter in embedded systems. They are used for short distance, wired communication between integrated circuits and microcontrollers. Since they are encountered frequently, you will need to have an understanding of how the protocols are implemented and how to use them. 

### SPI vs I2C
SPI requires at least 4 pins, with an additional chip select pin needed for each additional peripheral connected. It can support only 1 controller and an arbitrary number of peripherals, subject to the number of available chip select pins. This makes it well suited for high data rate, full-duplex (simultaneous send and receive) transmissions, supporting a clock speed of up to 10MHz. 

![SPI connection diagram](https://cdn.sparkfun.com/r/600-600/assets/learn_tutorials/8/2/SPI-Block_Diagram_Updated.png) 

I2C requires only two pins, allowing for up to 1008 peripheral devices. It also support a multi controller system, although the controllers are unable to communicate with each other and cannot use the bus simultaneously. Data rates for I2C fall between 100kHz-400kHz.  

![I2C connection diagram](https://cdn.sparkfun.com/r/600-600/assets/learn_tutorials/8/2/I2C-Block-Diagram.jpg)
