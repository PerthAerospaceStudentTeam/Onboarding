# HAB Telemetry Simulator
Develop an understanding of how telemetry data is generated, structured and interpreted in aerospace systems by simulating realistic high-altitude balloon (HAB) flight data.

## Background
Telemetry is the process of collecting and transmitting data from a remote system. In aerospace applications, telemetry allows engineers to monitor the state of a vehicle in real time, including its position, motion and health.

For a high-altitude balloon (HAB), telemetry typically includes measurements such as altitude, acceleration, temperature and system status. These values are obtained from onboard sensors, which are inherently imperfect. They contain noise, bias and occasional faults.

In this project, you will simulate a HAB flight profile (ascent, float, burst, descent) and generate realistic telemetry data. The focus is not on perfect physical modelling, but on producing **plausible, consistent and interpretable data** that reflects the behaviour of real HAB systems.

This project builds foundational skills in:
- Data modelling and simulation
- Sensor behaviour (noise, bias, drift)
- Time-series data handling
- Visualisation and interpretation

These skills are directly relevant to PAST’s work in HAB missions and future CubeSat telemetry systems.


## Project Deliverables

- Develop a Python script that generates telemetry data over time for a HAB flight. Your dataset must include:
  - Time
  - Altitude
  - Velocity
  - Acceleration
  - Simulated barometer altitude (with noise)
  - Simulated accelerometer readings (with noise and bias)

- Implement a flight profile that includes:
  - Ascent phase (increasing altitude)
  - Float phase (approximately constant altitude)
  - Descent phase (decreasing altitude)

- Ensure the generated data is **physically consistent**:
  - Velocity should correspond to changes in altitude
  - Acceleration should correspond to changes in velocity

- Add sensor characteristics:
  - Barometer: Gaussian noise
  - Accelerometer: Gaussian noise + constant bias

- Export the generated telemetry to a `.csv` file

- Produce plots showing:
  - True vs simulated altitude
  - Velocity vs time
  - Acceleration vs time

- Provide a short written explanation (~300–500 words) covering:
  - Why barometer data is noisy but stable
  - Why accelerometer data drifts over time
  - What makes your telemetry “realistic”


## Extra Deliverables

- Introduce at least one anomaly into your dataset, such as:
  - Sensor spike
  - Data dropout
  - Gradual sensor drift

- Allow user-defined parameters for:
  - Noise levels
  - Bias values
  - Flight duration

- Improve realism of the flight profile:
  - Smooth transitions between ascent, float, and descent
  - More natural acceleration behaviour
