# Power System Monitor
The Electrical Power System (EPS) is one of the most critical subsystems on any spacecraft. If power fails, nothing else works and your craft becomes little more than expensive space junk. The EPS monitor project will provide the opportunity for you to learn how to design a telemetry data pipeline, parse structured data packets, detect anomalies, and respond to fault conditions. All skills that translate to developing flight software for the CubeSat.

## Background
The EPS manages solar power input, battery charging and discharging, and power distribution to all other subsystems. The flight computer needs to monitor the state of the EPS and respond appropriately when required. This can include: 
- Reducing power to non-essential systems when the battery is low.
- Triggering safe mode if voltage drops below a critical threshold.
- Logging anomalies for post-mission analysis.

To streamline the development process, software can be designed and tested before hardware is available. To do this you will need to setup a simple program that can stream simulated data inputs to the system being tested, mimicking the expected values from the real hardware. 

Before any code is written, it is best practice to define an Interface Control Document (ICD) specifying what data the EPS transmits, in what format, and at what rate. This is a crucial step in any project that involves multiple independent teams developing an interconnected system. 


### Group Work
This project can be split between two people. If doing this, one person will be responsible for the EPS monitor while the other is in charge of the telemetry simulator. You will both work together to outline the packet structure in the ICD and design each half to be compatible with your specification. 

## Project Deliverables
- An ICD defining the packet structure for EPS telemetry. This should include:
    - The following data fields at minimum:
        - Bus voltage 
        - Current draw
        - Battery charge level 
        - Battery temperature 
    - Units associated with each field 
    - Packet transmission rate 
    - Packet framing scheme
- EPS simulator. This is a script that can generate EPS telemetry packets and transmit them over serial, in line with the specification outlined in your ICD. The simulator should include a command to inject a fault scenario (low battery, over- temperature, voltage drop) into the data stream. 
- EPS monitor. An application that can receive and parse the generated EPS telemetry packets. Should display the current state of the EPS system, log all received data to a file and alert when values cross a critical threshold. 
- Documented tests demonstrating how the monitor responds to expected input and fault scenarios.

## Stretch Goals 
- Add a dashboard that displays live EPS telemetry graphically, updating in real time as packets arrive
