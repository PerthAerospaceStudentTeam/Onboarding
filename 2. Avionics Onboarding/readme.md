# Avionics Onboarding
The Avionics Department specializes in designing, developing and testing electronic systems. This could involve:
- Electrical power systems
- Radio communications
- and Microcontrollers, actuators & sensors 
An Avionics member should know basic PCB design, electrical & electronics, and the theory behind the subsystem they choose work on.

Avionics Onboarding Projects range in guidance level. Some are more guided to help build your skills, while others are more open ended, giving you the opportunity to research and design your own system. If you end up becoming a PAST member, your work would likely be a continuation of the research and concept development you do in Onboarding.


> [!IMPORTANT]
> If you would like to become an Avionics member, there must be evidence Avionics Onboarding Project/s in your logbook. (There is an exception for the CAP subsystem)


# Intro to the Avionics Department
This is a resource for recruits to get an idea into the different divisions within the avionics department, and to help recruits choose an onboarding project based on their interests.

Avionics can be divided into three different subsystems.
```
Avionics
├── Electrical Power System (EPS)
├── Communication and Radio Frequency (STARCOMM)
└── Command And Processing (CAP)
```
## Electrical Power System (EPS)
The EPS subsystem works on the power generation, storage and distribution for the CubeSat. 
If you are interested in:
- Power generation: Maximum Power Point Tracking project would be the most suitable project
- Power storage: Buck Converter PCB & Power Inhibits Circuit would be the most suitable project
- Power Distribution: Power Distribution Safety would be the most suitable project

## Communication and Radio Frequency (STARCOMM)
The STARCOMM subsystems works on GPS and communicating with the CubeSat during flight through using radio.
You should join STARCOMM if you have an interest in RF components (Radio Frequency). 
Suitable projects for STARCOMM are:
- Radio Dipole Antenna
- Cubesat Radio Demodulation

## Command And Processing (CAP)
The CAP subsystems works on the electronic parts of the flight computer of the CubeSat.
This is a smaller subsystems because the large majority of the work for flight computer is software oriented. 
If you are interested in joining the CAP subsystems, suitable projects are:
- Secondary Flight Storage (Avionics)
- Device Drivers (Software)
- RTOS (Software)
- Finite State Machine (Software)

> [!IMPORTANT]
> You may still be considered for an Avionics member if you work on one of the software projects mentioned above.
